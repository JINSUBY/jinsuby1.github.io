## 이진섭	pytorch-cnn-visualizations OSS

&nbsp;&nbsp;&nbsp;&nbsp; pytorch-cnn-visualizations OSS 프로젝트는 파이토치를 Base로 하여 CNN 모델을 해석적으로 보여줄 수 있게하는 Visualization Tool 이다.
최근 딥러닝 모델 프로그래밍을 간소화 하기위한 Library 들이 많이 등장하고 있지만, Black Box인 딥러닝 모델을 해석적으로 Visualization 할 수 있는 Tool이 마땅치 않아
CAM, Grad-CAM 등의 코드를 직접 구현하여야 한다. 딥러닝 모델이 이미지의 어떠한 local feature map을 보고 prediction을 하는지 결정하는데 이러한 해석적인 결과가 중요하며
따라서 이러한 기능이 잘 구현이 된다면 User들이 이러한 Tool을 활발히 이용할 것이라 생각이 들었다.
현재 이 프로젝트는 대략 4800개의 스타를 받았다.
최근 commit한 Tool 중에는 Gradient visualization with vanilla backpropagation, Smooth grad, CNN filter visualization Inverted image representation과 Integrated gradients 등이 있다.
이 프로젝트를 통해 기여할 수 있는 바는 가장 대표적인 VGG, ResNet 이외에도 mobileNet, Efficient Net 등의 다양한 딥러닝 Backbone에서 이러한 Tool들을 구현해나가면
이용자의 모델에 맞게 Tool을 사용할 수 있을 것이다.


## 임호진	tinykaboom
&nbsp;&nbsp;&nbsp;&nbsp; 
이 tinykaboom이라는 오픈소스는 컴퓨터 그래픽 강의 자료로 만들어진 것으로 보이는데, 실행하게 될 경우 구체가 폭발하는 한 이미지를 생성하게 된다. Wiki를 보면 폭발하는 이미지를 만들기 위한 과정이 상세하게 설명되어 있다. 우선 이 프로젝트를 선택한 이유를 설명하자면 폭발하는 이미지를 만드는 과정에 흥미가 동하기도 했고, 완성도가 낮은 상태로 방치된 것으로 보였기 때문이다. 다른 프로젝트의 경우 조금 복잡하기도 하고 이미 많은 사람들이 contribute해서 대부분 최근 commit이 버그를 고치거나, readme파일을 수정하는 정도인 경우가 많았다. 하지만 이 프로젝트에 contribute하게 된다면 눈에 보이는 변화를 만들 수 있을 것 같다는 점이 재미있을 것 같았다. 이 프로젝트의 경우 전부를 통틀어서 commit한 내용은 작성자의 주석 추가, readme수정이였다. 그 내용들도 살펴보고, 프로그램도 실행해보면서 두가지 기여 방식을 생각하였다. 첫번째는 직접 폭발하는 이미지를 만드는 일련의 과정을 경험하면서 배우는 입장으로 처음 보는 사람도 이해하기 쉽도록 주석을 추가하는 것이다. 아무래도 가르치는 입장에서는 배우는 사람이 이미 알고 있다고 생각할 수 있기 때문에 그런 부분에서 기여를 할 수 있다고 보여진다. 두번째는 성능을 확장시키는 것이다. 해당 Repository에는 gif, mp4, jpg형식의 파일들이 각각 존재하는데, 폭발하는 짧은 영상들도 포함되어있다. 하지만 실제로 프로그램을 실행시키면 ppm이라는 이미지 파일만 나오게 되는데, 여러 번 실행을 해도 같은 이미지만 나오게 된다. 이 부분을 다른 형식을 생성하게 하거나, 실행할 때마다 폭발하는 시점이 다른 랜덤한 이미지를 생성하도록 만들 수 있다고 생각한다.


## 이수민	FlatBuffer
&nbsp;&nbsp;&nbsp;&nbsp; 
FlatBuffers란 메모리 효율을 극대화하도록 설계된 교차 플랫폼 직렬화 라이브러리다.
FlatBuffers는 직렬화된 데이터를 상위 혹은 하위 호환성을 가진 채로 먼저 파싱하거나 언패킹하지 않고 바로 직렬화된 데이터에 접근할 수 있도록 한다. 
이 프로젝트를 선정한 이유는 메모리 효율을 극대화하는 라이브러리라는 점에서 흥미가 생겼고,
생각보다 참여가 활발하지 않아 팀 프로젝트로 참여할 수 있는 부분이 많이 있을 것이라는 생각이 들었기 때문이다. 
해당 프로젝트를 rust 언어에서 지원하는 부분의 개발이 아직 부족한 상태이므로 여기에 참여하여 rust에서 FlatBuffers를 사용할 수 있도록 하는 것을 계획하고 있다.


## 김민제	pysnoop
&nbsp;&nbsp;&nbsp;&nbsp; 
pysnoop는 python 파일을 컴파일 할 때, 어느 줄이 컴파일이 되었는지, 어느 줄에서 이상이 생겼는지 보여주는 디버깅 툴 소프트웨어이다.
여기서 pysnoop만의 특별한 점은, 출력 결과물에 시간, 변수,배 값들, 코드 내용이 포함되어 있어서 초보자 빠르고 쉽게 파악할 수 있다는 장점이 있다.
위 프로젝트는 대략 14000의 star와 800개 이상의 fork를 받은 인기있는 프로젝트이다. 
최근 커밋으로는 python 3.8이 출시되어서 python3.8버전이 추가되었고, 속도개선, 다양한 버그 수정등이 있었다. 최근 이슈로는 Time 버그, 원하는 범위 만의 출력 등의 성능 개선들이 존재한다.
다양한 소프트웨어 툴들이 디버깅 기능을 제공하고 있지만 이들은 한눈에 보기가 쉽지 않고, 읽기 불편한 점들이 있습니다. 수정해보기 위해 작성한 코드에서 버그가 발생해 해결하기 위해서 디버깅 모드를 켰으나, 어떻게 사용하는지 어려워서 포기했던 적이 많다.
이에 pysnoop는 코드를 포함하여 시간순으로 나열해서 보여주니까, 어느 부분이 실행될 것이고, 해당 줄에 변수값도 나오니, 상황파악과 읽기가 훨씬 쉬워진다.
따라서, 본 프로젝트의 목적에 맞게 pysnoop가 어떻게 하면, 사용자들이 더 쉽게 읽을 수 있을지, 버그는 어느 부분에 있고, 불필요할 때 디버깅 결과물이 출력되지 않고 특정 조건에서만 작동되도록 하는 등과 관련된 개선을 하고자 한다.


## 천기오	Flutter
&nbsp;&nbsp;&nbsp;&nbsp; 
Flutter는 구글이 개발한 오프 소스 모바일 애플리케이션 개발 프레임워크인 플러터의 기능들을 연습할 수 있도록 샘플을 모아 놓은 프로젝트이다.
최근 플러터를 가지고 앱을 개발한 적이 있었는데 개발 이전에 학습을 하면서 참고할만한 자료가 상당히 부족하다는 느낌을 받은 적이 있었다. 따라서 이 프로젝트가 처음 플러터를 접하는 사람들에게 많은 도움이 될 것이라고 생각하여 흥미를 느꼈다.
이슈를 보면 대부분 샘플들의 개선 방향을 제시하는 이슈들이 대부분이다. 따라서 나 또한 여러 샘플들을 사용해보고 개선할 점을 리포트하고, 간단한 부분은 직접 수정하거나 버그 픽스도 하며 기여를 할 계획이다.


## 양세중	Lizard
&nbsp;&nbsp;&nbsp;&nbsp; 
Lizard는 C / C ++를 포함한 많은 프로그래밍 언어를 위한 확장 가능한 Cyclomatic Complexity Analyzer 이다. (모든 헤더 파일 또는 Java 가져 오기가 필요하지 않음). 또한 복사-붙여 넣기 감지 (코드 복제 감지 / 코드 중복 감지) 및 기타 여러 형태의 정적 코드 분석을 수행한다. 현재 대략 1000개의 스타를 받았다.
최근 이슈로는 Python Fuction에서 nloc이 부정확한 경우가 있다는 점, js 함수를 과소보고 등이 있다. 특정 기능을 하는 함수에서 부정확한 결과가 나온다는 점 등 몇몇 버그가 있는 걸로 보아, 수정해야 할 부분이 있다고 생각한다. Openhub에서 찾아본 결과, 총 코드는 54.1K이고 마지막 commit은 18day로 Low Activity라고 할 수 있다.
코드를 수정하는 것은 쉽지 않겠지만, 아직 결과가 정확하지 않고, 모든 언어를 지원한다는 점에서 오류가 있는 함수를 만들어 돌려보고, 이슈를 제기하는 것만으로도 코드 향상에 크게 기여할 수 있다고 생각한다. 코드를 들여다보고 어디에서 오류가 나는지 제보하거나 주석을 고치는 방향으로 develop하고 싶다. 그리고 warning이 나는 곳도 있다고 나오는데, 가능하면 그 부분도 수정할 수 있으면 한다.
